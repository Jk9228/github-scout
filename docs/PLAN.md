# GitHub 偵察機 - 開發計畫

## 流程解說（像寄包裹一樣）

```
[你在輸入框打字] → [按下搜尋按鈕] → [JavaScript 記住你打的字] 
    → [發送網路請求] → [GitHub 伺服器收到要求] 
    → [伺服器找到資料] → [傳回資料] → [網頁顯示結果]
```

### 第一步：寫信（輸入框打字）
你在輸入框打「octocat」，就像在便利貼上寫「我想找這個人」。

### 第二步：封信（按下按鈕）
按下搜尋按鈕，就像把便利貼交給快遞先生。

### 第三步：快遞出發（網路請求）
JavaScript 把你的要求包成一份資料，透過網路寄給 GitHub。

### 第四步：圖書館管理員收到（伺服器處理）
GitHub 的伺服器收到你的要求，去資料庫裡找這個人。

### 第五步：快遞回來（回應資料）
伺服器把找到的資料（名字、照片、統計資料）傳回來。

### 第六步：打開信封（顯示結果）
網頁把傳回來的資料顯示在畫面上，你就能看到結果了！

---

## 實作步驟

### Step 1: 做出輸入和按鈕的畫面
- 建立搜尋框和按鈕
- **目標**：能在框框裡打字，按下按鈕

### Step 2: 讓程式能「記住」打的名字
- 建立一個「盒子」來存放使用者打的字
- **目標**：點按鈕後，能看到剛才打了什麼

### Step 3: 發送網路請求到 GitHub
- 用 fetch 呼叫 GitHub API
- **目標**：拿到 GitHub 使用者的資料

### Step 4: 把結果顯示在畫面上
- 把拿到的資料變成卡片顯示
- **目標**：看到使用者的頭像和名字

### Step 5: 加入 Loading 狀態
- 搜尋時顯示「載入中...」
- **目標**：讓使用者知道系統正在工作

### Step 6: 加入錯誤處理
- 找不到使用者時顯示錯誤訊息
- **目標**：使用者打錯名字時不會當掉
